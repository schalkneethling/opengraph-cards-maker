# 11ty

The most reliable 11ty integration is a build-time Node script:

1. Read the same content data your templates use.
2. Generate PNG files into your Eleventy input or output tree.
3. Reference each generated image from your layout metadata.

This keeps Open Graph images static and works with any Eleventy deploy target.

## Install

```bash
pnpm add -D opengraph-cards-maker
pnpm exec playwright install chromium
```

## package.json

Run the generator after Eleventy unless the build itself needs the generated
files.

Running after Eleventy is the simplest default. It works well when the generator
reads files from `_site` or writes directly into `_site`:

```json
{
  "scripts": {
    "build": "npx @11ty/eleventy && node scripts/generate-og-cards.js"
  }
}
```

Run the generator before Eleventy only when Eleventy needs those files during
its own build, such as when `addPassthroughCopy()` copies generated images from
the input directory into `_site` or when templates read generated metadata:

```json
{
  "scripts": {
    "build": "node scripts/generate-og-cards.js && npx @11ty/eleventy"
  }
}
```

## Frontmatter Script

For sites with frontmatter-based content, generate cards directly from file
metadata:

```js
import path from "node:path";
import { createCardsFromContentFiles, generateOpenGraphCards } from "opengraph-cards-maker";

const outputDir = path.resolve("src/assets/og/posts");

const cards = await createCardsFromContentFiles({
  contentDir: "src/posts",
  extensions: [".md", ".njk"],
  eyebrow: "example.com",
  getMeta: ({ frontmatter }) => [{ label: "Published", value: String(frontmatter.date) }],
});

await generateOpenGraphCards({
  outputDir,
  cleanOutputDir: true,
  cards,
});
```

Configure Eleventy to pass the generated images through:

```js
export default function (eleventyConfig) {
  eleventyConfig.addPassthroughCopy("src/assets/og");
}
```

With this output path, run the generator before Eleventy so the generated images
exist before passthrough copy runs.

For richer data, build card objects from your own collections or data files and
pass them directly to `generateOpenGraphCards()`.

## Generic Generated JSON

If your site already emits one JSON file per page, read those files and pass them
to the generator. This is common when Eleventy also generates page data for
search indexes, client-side navigation, or API-like endpoints.

The script only needs to map each page object into the card data shape:

```js
import fs from "node:fs/promises";
import path from "node:path";
import { generateOpenGraphCards } from "opengraph-cards-maker";

const pagesDir = "_site/page-data";
const files = await fs.readdir(pagesDir);

const cards = await Promise.all(
  files
    .filter((file) => file.endsWith(".json"))
    .map(async (file) => {
      const page = JSON.parse(await fs.readFile(path.join(pagesDir, file), "utf8"));

      return {
        id: page.slug,
        eyebrow: page.section,
        title: page.title,
        description: page.description,
      };
    }),
);

await generateOpenGraphCards({
  outputDir: "_site/assets/og/pages",
  cleanOutputDir: true,
  cards,
});
```

## Template Metadata

Add the generated image URL to your layout:

```njk
<meta property="og:image" content="{{ site.url }}assets/og/posts/{{ page.fileSlug }}.png">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta name="twitter:card" content="summary_large_image">
```

Adjust the path and ID to match the script that writes your card files.
